Les composants EniBook sont des éléments HTML personnalisés (custom element) représentés par des balises du type <element-name-it></element-name-it> dont le nom se termine toujours par le suffixe -it.

Les développements sont réalisés en typescript et sont accessibles depuis l’API (Application Programming Interface).

1. Bascule

Fonction Elément html Classe typescript

Cet élément propose un bouton pour montrer ou cacher un autre élément html défini par son sélecteur css.

<toggle-it>

type : « en ligne » (inline-level element)

ToggleIt
Structure

Il s’agit d’un simple bouton qui affiche dans une infobulle l’action qui lui est associée : montrer ou cacher un autre élément.

Interactivité

Cet élément est cliquable mais non éditable .

L’interaction est de type bascule (toggle) et consiste à montrer ou cacher un autre élément : si cet élément est caché alors le clic sur le bouton conduit à le montrer, sinon il le cache.

1.1. Attributs et propriétés

Cette balise possède 7 attributs spécifiques optionnels : selector, size, text-hide, text-show, tooltip-hide, tooltip-show et visibility.

Le premier exemple définit ces 7 attributs :

<div class="flex flex-justify-between">
  <toggle-it id="toggle-left" visibility selector="#toggle-right" text-show="G" text-hide="G" tooltip-show="Montrer D" tooltip-hide="Cacher D" size="large"></toggle-it>
  <toggle-it id="toggle-right" visibility selector="#toggle-left" text-show="D" text-hide="D" tooltip-show="Montrer G" tooltip-hide="Cacher G" size="large"></toggle-it>
</div>

1.1.1. Sélecteur css

L’attribut selector (attribut obligatoire) définit l’élément html à cacher ou à montrer à l’aide d’un sélecteur css.

Un sélecteur css est une chaîne de caractères qui définit le (ou les) élément(s) sur lesquels s’applique un ensemble de règles css [1]. Si plusieurs éléments répondent au même critère, seul le premier d’entre-eux est concerné par la bascule.

1.1.2. Visibilité

Par défaut, cacher un élément signifie le supprimer de la page : la mise en page du document est ainsi potentiellement modifiée.

L’attribut visibility peut être utilisé afin de cacher un élément tout en conservant occupé l’espace dans lequel il aurait été visible : la mise en page du document reste ainsi identique qu’il soit visible ou non, contrairement au fonctionnement par défaut.

Sans l’attribut visibility

<toggle-it selector="…​"></toggle-it>

début fin
Avec l’attribut visibility

<toggle-it visibility selector="…​"></toggle-it>

début fin

1.1.3. Textes du bouton

Deux attributs text-hide (défaut : ) et text-show (défaut : ) permettent de définir le texte du bouton respectivement quand l’élément est caché ou non.

Fonctionnement par défaut

<toggle-it selector="…​"></toggle-it>

début fin
Adaptation des textes

<toggle-it selector="…​" text-show="…​" text-hide="…​"></toggle-it>

début fin

1.1.4. Infobulles

Deux attributs tooltip-hide (défaut : Cacher selector) et tooltip-show (défaut : Montrer selector) permettent de définir le texte de l’infobulle respectivement quand l’élément est caché ou non.

Fonctionnement par défaut

<toggle-it selector="…​"></toggle-it>

début fin
Adaptation des infobulles

<toggle-it selector="…​" tooltip-show="…​" tooltip-hide="…​"></toggle-it>

début fin

1.2. API

L’API (Application Programming Interface) de l’élément <toggle-it> est consultable ici.

L’API en bref
Usage en typescript 
import { ToggleIt } from '@enibook/elements/dist/elements/toggle/toggle' (1)
const toggle = new ToggleIt()                                            (2)
toggle.selector = "#ident"                                               (3)
document.body.appendChild(toggle)                                        (4)
/*
<body>
...
<toggle-it selector="#ident"></toggle-it>
</body>
*/
1 importer la classe ToggleIt;
2 créer une instance de cette classe;
3 préciser le sélecteur de l’élément que l’instance doit cacher ou montrer;
4 ajouter l’instance dans le DOM.

2. Editeur de code

Fonction Elément html Classe typescript

Cet élément est un éditeur de code source.

<code-it>

type : « bloc » (block-level element)

CodeIt
Structure

La fenêtre d’édition est encadrée au-dessus par une barre d’outils et en-dessous par une barre d’informations. Ces deux barres sont escamotables.

Interactivité

Cet éditeur de code est évidemment cliquable et éditable .

Partie gauche de la barre d’outils

annuler toutes les modifications et revenir au code initial

annuler la dernière modification

rétablir la dernière annulation

indenter les lignes sélectionnées du nombre d’espaces correspondant à une tabulation (2 espaces par défaut)

désindenter les lignes sélectionnées du nombre d’espaces correspondant à une tabulation (2 espaces par défaut)

commenter/décommenter ligne par ligne les lignes sélectionnées

commenter/décommenter le bloc de lignes sélectionnées
Partie droite de la barre d’outils

affiche le logo du langage courant et propose un menu déroulant pour choisir le langage source de l’éditeur parmi les langages reconnus

menu déroulant pour activer une commande de l’éditeur et indiquer le raccourci clavier associé

afficher ou cacher les numéros de ligne

copier le contenu de l’éditeur dans le presse-papier

changer de thème : passer de clair à sombre ou inversement

passer l’éditeur en mode plein écran

quitter le mode plein écran

2.1. Attributs et propriétés

Cette balise possède XXX attributs spécifiques optionnels : date, size et time.

2.2. API

L’API (Application Programming Interface) de l’élément <code-it> est consultable ici.

L’API en bref
Usage en typescript 
import { CodeIt } from '@enibook/elements/dist/elements/code/code' (1)
const editor = new CodeIt()                                           (2)
editor.size = 'large'                                                  (3)

...

*/

5 ajouter l’instance dans le DOM.

3. Horloge

Fonction Elément html Classe typescript

Cet élément affiche une horloge numérique indiquant l’heure et la date courantes.

<clock-it>

type : « en ligne » (inline-level element)

ClockIt
Structure

  • A gauche de l’icône , la date est affichée sous la forme jour quantième mois année[2] où jour est le nom du jour (lundi, mardi…​), quantième le numéro du jour dans le mois (1, 2…​), mois le mois en toutes lettres (janvier, février…​) et année l’année en chiffres (exemples: jeudi 8 juin 2023, mercredi 5 avril 2034).

  • A droite de l’icône , l’heure est présentée sous la forme hh:mm:ss où hh représente l’heure avec deux chiffres, mm les minutes avec deux chiffres et ss les secondes également avec deux chiffres (exemples: 08:12:54, 13:02:05).

Interactivité

Cet élément est cliquable mais non éditable . L’interaction consiste à adapter l’affichage de l’horloge aux besoins de l’utilisateur.

L’horloge est un bouton cliquable à 4 états. En cliquant sur l’horloge, on passe successivement dans les états suivants :

  1. affichage de l’heure uniquement :

  2. affichage de la date et de l’heure :

  3. affichage de la date uniquement :

  4. affichage de l’icône uniquement :

3.1. Attributs et propriétés

Cette balise possède 3 attributs spécifiques optionnels : date, size et time.

3.1.1. Taille de l’horloge

L’attribut size (small par défaut, medium ou large) précise la taille de l’horloge.

3.1.2. Réglage de l’affichage

  • L’attribut date permet d’afficher la date.

  • L’attribut time permet d’afficher l’heure.

3.2. API

L’API (Application Programming Interface) de l’élément <clock-it> est consultable ici.

L’API en bref
Usage en typescript 
import { ClockIt } from '@enibook/elements/dist/elements/clock/clock' (1)
const clock = new ClockIt()                                           (2)
clock.size = 'large'                                                  (3)
clock.date = true                                                     (4)
document.body.appendChild(clock)                                      (5)
/*
<body>
...
<clock-it size="large" date></clock-it>
</body>
*/
1 importer la classe ClockIt;
2 créer une instance de cette classe;
3 préciser la taille de l’instance;
4 préciser le type affichage désiré;
5 ajouter l’instance dans le DOM.

4. Lorem Ipsum

Fonction Elément html Classe typescript

Le Lorem Ipsum permet d’illustrer une mise en page avec différents gabarits contenant du texte vide de sens[3].

3. Un texte lisible et contenant du sens serait source de distractions et empêcherait de se concentrer sur la mise en page elle-même.

<lorem-ipsum-it>

type : « bloc » (block-level element)

LoremIpsumIt
Structure

Le Lorem Ipsum a simplement la structure du texte qu’il représente : mot, phrase, paragraphe, item de liste.

Interactivité

Cet élément n’est pas interactif : il est non cliquable et non éditable .

4.1. Gabarits

L’élément <lorem-ipsum-it> peut générer 6 types de gabarits différents.

Gabarit Paramètre correspondant Exemple

mot

word

phrase

sentence

paragraphe

paragraph

item de liste à puces

ulist

item de liste ordonnée

olist

item de liste de descriptions

dlist

4.2. Attributs et propriétés

Cette balise possède 2 attributs spécifiques optionnels :

  • type : le gabarit des objets textuels à générer, à savoir : word, sentence, paragraph (valeur par défaut), ulist, olist ou dlist;

  • count : le nombre d’objets textuels à générer (valeur par défaut: 1).

    Quatre mots

    Deux phrases

    Un paragraphe

    Une liste à puces de 2 items

    Une liste énumérée de 3 items

    Une liste de descriptions de 2 items

4.3. API

L’API (Application Programming Interface) de l’élément <lorem-ipsum-it> est consultable ici.

L’API en bref
Usage en typescript 
import { LoremIpsumIt } from '@enibook/elements/lorem-ipsum/lorem-ipsum' (1)
const lorem = new LoremIpsumIt()                                         (2)
lorem.type = 'dlist'                                                     (3)
lorem.count = 3                                                          (4)
document.body.appendChild(lorem)                                         (5)
/*
<body>
...
<lorem-ipsum-it type="dlist" count="3"></lorem-ipsum-it>
</body>
*/
1 importer la classe LoremIpsumIt;
2 créer une instance de cette classe;
3 préciser le type de gabarit de l’instance;
4 préciser le nombre d’objets textuels à générer par l’instance;
5 ajouter l’instance dans le DOM.
